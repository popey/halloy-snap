# Contributing

Thanks for wanting to contribute! A few small guidelines to keep contributions welcome and reviewable:

- Keep changes small and focused (one concern per PR).
- Avoid adding new dependencies without opening an issue first.
- Run CI locally where possible and include testing notes in your PR description.
- Provide motivation and short reproduction steps for bug fixes.
- For docs-only changes, a short description and link to any related issue is sufficient.

If you're new here, say hi in an issue and we’ll help you find a good first task. See README for project-specific setup.
