# Contributing

Thanks for wanting to contribute to halloy-snap! A few simple guidelines:

- Keep PRs small and focused (one logical change per PR).
- Avoid adding new dependencies without opening an issue first.
- Describe the motivation and testing steps in the PR body.
- Run CI locally where possible (see .github/workflows). If CI fails due to infra, note that in the PR.
- Be respectful and constructive in code review discussions.

If you're new, open an issue describing what you'd like to work on and a maintainer will help.
